# 赔率专项查询（场景 F）

## 何时使用

用户关注**盘口本身**，而非完整比赛分析：

| 典型问法 | 意图 |
|----------|------|
| 「这场赔率对比一下」 | 多家公司当前赔率横向快照 |
| 「赔率有变化吗」 | 初盘到临场走势追踪 |
| 「今天哪些比赛赔率变动最大」 | 先拉彩票赛程 → 按变动排序 |
| 「凯利指数怎么看」 | 独立凯利查询 |

**不要**在用户仅问完整分析时单独走本 Skill；深度分析内已内嵌盘口维度。

## 查询路径消歧

| 用户意图 | 工具路径 | 说明 |
|----------|----------|------|
| 当前赔率横向对比 | `get_odds_snapshot` | 同一时刻欧赔/亚盘/大小球快照 |
| 赔率随时间变化 | `get_odds_trend` | 初盘→临场纵向变动 |
| 按变动幅度排名 | `get_lottery_schedule` + 各场 `get_odds_trend` | 需先知道当日可售场次 |
| 凯利指数 | `get_kelly` | 用户显式追问时使用 |
| 同赔历史 | `get_same_odds_history` | 统计回归参考 |
| 必发资金 | `get_betfair` | 资金分布参考 |

## 处理流程

```
1. 若涉及具体比赛 → 先定位 match_id
2. 按意图选择单一或组合盘口工具（避免无意义全量查询）
3. 归纳对比结果，突出用户关心的维度
4. 不做完整六段式分析（除非用户追加「全面分析」）
```

## 输出要点

- **快照对比**：主胜/平局/客胜或让球盘口、水位，注明数据来源时刻感
- **走势追踪**：初盘→现盘变化方向与幅度，辅助判断买入时机（不做投注承诺）
- **变动排名**：列表形式，变动最大场次优先
- **凯利/同赔/必发**：独立段落，解释指标含义 + 本场数值

## 让球表述

- 使用数据库中文表述（`line_cn`），**禁止** LLM 自行换算盘口方向

## 盘口不足

- 欧赔 + 亚盘均缺失：标注「盘口数据不足，建议跳过此场」
- 部分缺失：按 `foretell-status-dictionary` 标注，不编造

## 购彩语境

- 纯赔率查询**默认不加**购彩风险提示
- 用户结合赔率问「怎么买」「推荐方向」→ 切换 `foretell-betting-single` 并加风险提示

## 输出纪律

遵守 `foretell-output-discipline`：不暴露工具名、不输出中间态。
