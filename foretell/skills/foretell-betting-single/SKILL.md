# 单场购彩推荐（场景 D）

## 何时使用

用户带有**明确投注倾向**的单场决策：

| 典型问法 | 说明 |
|----------|------|
| 「这场怎么买」 | 单场投注方案 |
| 「主不败可以吗」 | 玩法倾向确认 + 方案 |
| 「给个单场方案」「赛事方案」 | 结构化购彩输出 |

**不要**与场景 A 轻量查询混用；也不要在批量扫盘初筛阶段直接作为最终推荐（见场景 E）。

## 处理流程

```
1. 定位比赛（entity-resolver 或 resolve）
2. 走深度分析路径：并行委派 fundamentals / odds / intel 三维度子智能体
3. 综合六段式分析，形成**单一明确主推方向**
4. 生成投注方案（玩法 + 方向 + 简要依据）
5. 结尾强制附加风险提示
```

## 推荐一致性规则（强制）

- 同一场、同一玩法（胜平负、让球、大小球等）**只能有一个明确主推方向**
- **严禁**「主不败 + 客不败」等互相矛盾的双向推荐
- 冷门或抢分可能仅在风险提示中用**中性语气**描述，禁止「推荐买入」「建议重仓」等投注语气
- 篮球使用胜负、让分、大小分术语，禁用足球「胜平负」

## 输出结构

1. **主推方案**（置顶）：玩法 + 单一方向 + 1–3 条核心依据
2. **简要分析摘要**：从六段分析中提炼与主推相关的要点（不必完整六段）
3. **风险与备选说明**：不利因素、冷门提示（中性语气，非第二主推）
4. **强制结尾**：`⚠️ 彩票有风险，投注需谨慎`

## 与场景 B 的关系

| 场景 B | 场景 D |
|--------|--------|
| 预测与分析为主 | 决策辅助，必须有明确主推 |
| 可并列多种倾向 | 单一主推，矛盾禁止 |
| 购彩风险提示视语境 | **每次必须**加风险提示 |

## 数据诚实

- 盘口不足（SKIP_MATCH）时：不建议强行推荐，说明「盘口数据不足，建议跳过此场」
- 数据缺失维度降权，不编造赔率支撑主推

## 输出纪律

遵守 `foretell-output-discipline`：不暴露工具名、不输出中间态。
