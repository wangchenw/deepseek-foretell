# 批量初筛与深核（场景 E）

## 何时使用

用户需要对**多场比赛**同时判断：

| 典型问法 | 说明 |
|----------|------|
| 「今天竞彩四串一怎么配」 | 多场合集 + 串关方案 |
| 「帮我扫盘」 | 批量初筛候选表 |
| 「十四场给个方案」「本期任九推荐」「北单扫盘」「竞彩篮球串关」 | 按玩法与期号/日期拉取场次列表 |

## 两阶段策略

```
第一阶段：get_lottery_schedule 拉取当日/当期彩票赛程（101/201/301/401/402/403/404）
    ↓
第二阶段：对每个场次 task(screening-agent) 批量初筛
    ↓
第三阶段（按需）：用户要求详细分析 → 对入选场次走深度路径（场景 B/D）
    ↓
第四阶段：生成串关/方案 → 附加风险提示
```

## 初筛 vs 深核

| 阶段 | 产出 | 能否作为最终推荐 |
|------|------|------------------|
| 批量初筛（screening-agent） | 候选场次、初筛方向、置信度 | **不能**直接作为最终推荐词 |
| 深度核验（三维度子智能体） | 完整分析、明确主推 | **可以**作为最终推荐 |

## 扫盘效率原则

- 用户只说「扫盘」「四串一」且未要求逐场展开 → **优先批量初筛**，不对每场走完整深度分析
- 用户明确「详细分析第 X 场」「展开讲某场」→ 补做深度核验
- 可并行发起多个 screening-agent task，控制并发合理即可

## 初筛与深核不一致时

- **以深度核验为准**
- 显式说明：「初筛方向为 X，深度核验后修正为 Y，依据：……」
- **禁止**把两套相反结论同时呈现给用户

## 输出结构（初筛阶段）

1. **候选场次表**：序号、对阵、初筛方向、置信度、一句话理由
2. **跳过场次**：盘口不足（SKIP_MATCH）或数据严重缺失的场次
3. **说明**：初筛结果供筛选参考，非最终投注建议
4. 若用户要方案/串关：在初筛基础上给出组合思路，结尾加风险提示

## 十四场 / 任九

- 玩法编码 `401`；用户未指定期号时使用当前在售期号
- 任九从十四场列表中选取 9 场，逻辑与十四场共用赛程
- 按期号查询：`get_lottery_schedule(play_type=401, period=期号)`
- 半全场、进球彩分别使用 `402`、`403`，北单使用 `301` / `404`，竞彩篮球使用 `201`

## 购彩护栏

涉及串关、方案、推荐时，结尾必须：`⚠️ 彩票有风险，投注需谨慎`

## 输出纪律

遵守 `foretell-output-discipline`：不暴露工具名、不输出「正在查询」。
