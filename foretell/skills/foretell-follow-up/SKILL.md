# 多轮追问（场景 G）

## 何时使用

用户在**同一会话**中基于上一轮分析继续提问：

| 典型问法 | 说明 |
|----------|------|
| 「比分预测」 | 具体比分区间与概率 |
| 「哪一队率先进球」 | 先进球概率判断 |
| 「各比分之间的概率」 | 比分矩阵细化 |
| 「上半场两队进球概率」 | 时段概率追问 |
| 「继续」「你刚才那段再优化」 | 延续或重写先前输出 |

## 核心机制

**会话上下文 = Checkpointer 对话历史**，不使用 match_context Store 键。

- 同 `thread_id` 内，最近对话自动可见
- 已完成的六段分析、已定位的 match_id 均从历史消息中获取
- **禁止**在追问轮次重复调用 entity-resolver（除非用户明确换场）

## 处理流程

```
1. 读取 Checkpointer 中最近对话（含六段分析、match_id 证据）
2. 确认用户追问类型（比分 / 先进球 / 时段 / 复述 / 优化）
3. 若需补充数据且历史 envelope 不足，可调用相关 Tool
4. 在既有分析基础上做细化推演，直接回答
5. 不重新走完整六段模板（除非用户要求「重新分析」）
```

## 常见追问链

```
用户：分析【竞彩足球周二004 巴黎 VS 拜仁】胜平负、比分
助手：[六段完整分析]
用户：比分预测
助手：[具体比分区间与概率，基于前述数据]
用户：哪一队率先进球
助手：[先进球概率分析]
```

## 输出要求

- **保持比赛定位稳定**：勿张冠李戴到其他场次
- 区分事实（来自 Tool/先前分析）与推断（概率推演）
- 数据不足时诚实说明，不凭印象编造赔率或比分
- 购彩相关追问结尾须含：`⚠️ 彩票有风险，投注需谨慎`
- 遵守 `foretell-output-discipline`：不暴露工具名、不输出「正在查询」

## 与场景 B 的分界

| 场景 B | 场景 G |
|--------|--------|
| 首次深度分析，走六段式 | 基于已有分析的细化追问 |
| 并行三维度子智能体 | 通常无需重新委派（除非缺数据） |
| 完整模板输出 | 针对性短答或段落补充 |

## 回溯更早轮次

若用户引用超出最近窗口的内容（如「第三段赔率明细」）：

- 优先从 Checkpointer 历史消息中查找
- 找不到时说明无法回溯，请用户复述关键信息
- 禁止凭 LLM 记忆编造未出现的赔率或数据
