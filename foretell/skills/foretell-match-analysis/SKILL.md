# 单场深度分析（场景 B）

## 何时使用

用户意图为**多维度整合后的专业判断**，而非轻量事实查询：

| 典型问法 | 说明 |
|----------|------|
| 「分析周二004 巴黎 VS 拜仁」 | 竞彩编号 + 深度分析 |
| 「谁会赢」「全面分析一下」 | 预测/模板分析 |
| 「按四步稳球模板分析」 | 结构化深度输出 |

**不要**与场景 A 轻量查询混用：用户问赛程列表、积分榜时走 `foretell-light-query`。

## 输出格式优先级

- 用户**明确指定**输出格式（如「按四步稳球模板分析」「只给比分预测」「列三点」）时，**优先用户格式**，不强制套六段式。
- 用户未指定格式时，**默认六段式**（篮球五段式）。
- 用户格式只覆盖部分段落时，未覆盖部分仍按六段式对应段落规范执行。

## 处理流程

```
1. 定位比赛（task → entity-resolver，或主智能体直接 resolve）
2. 并行委派三维度子智能体（单次 task 含完整 match_id 与球队 ID）：
   - fundamentals-analyst → 积分、近况、交锋
   - odds-analyst         → 欧赔、亚盘、大小球、走势
   - intel-analyst        → 阵容、伤停、情报
3. 收集各 task 返回的结构化 JSON
4. 按本 Skill 模板归纳最终输出
5. 写入 Checkpointer（供场景 G 追问）
```

## 子智能体委派要点

- 子智能体**无状态**：每次 task 须包含完整 match_id、team_id、指令
- 可并行发起三个 task，不必串行等待
- 子智能体返回 JSON，主智能体在上下文中汇总，**不落虚拟文件**

## 输出结构（足球默认六段）

### 结构硬约束

- 必须严格按下列六段**顺序与标题**输出，**不得自造段落标题**。
- 禁用负样本标题：「对阵基本面」「盘口数据」「情报维度」「综合倾向」「比分倾向」「结论」等均为禁止。
- 第①段必须**置顶**核心结论，禁止把结论放最后一段。
- 置信度用文字（**高 / 中 / 中低 / 低**），**禁用星级符号**（如 ★★★☆☆）。

### 六段内容

1. **核心结论与推荐**（4 行以内：胜平负倾向、让球倾向、大小球倾向、比分区间、置信度）— 必须置顶
2. **赛事基础信息概览**（时间、联赛、主客、场地）— 中立场须标注「中立场」
3. **赛程体能与环境因素** — 中立场赛事须剔除主场战绩权重，或明确说明「名义主场实为中立场」，**不得用主场战绩作核心支撑论据**
4. **伤停与战术权重** — 数据不足时按 `foretell-status-dictionary` 标注
5. **机构操盘与资金陷阱** — **必须讨论盘口陷阱可能性**（诱上 / 诱下 / 水位异常 / 满水诱盘），不得只下「强势盘」单向结论
6. **战术打法与比赛走势推演** — 比分倾向须基于攻防数据 + 盘口推导，**按概率排序**；禁止把胜平负方向（机构主推客胜）当比分方向（如「2-1 / 1-0 为机构主推」）混述

篮球使用五段式（见 Foretell 产品文档 §2.4）。

## 场景边界（强制）

- 本场景为**预测 / 分析**，**禁止**输出具体购彩玩法 + 赔率的投注方案（如「竞彩让 +1 客胜赔率 3.14」「周六069 让 +1 客胜」）。
- 购彩方案只在 `foretell-betting-single`（场景 D）下生成，且须用户明确问「怎么买 / 给方案 / 单场方案」。
- 本场景可给方向倾向（胜平负倾向、让球倾向、大小球倾向），但**不得**带具体玩法编号 + 赔率。

## 专业表达规范

- **欧赔隐含概率**须归一化后展示（三向总和回算到 100%），并注明返水率，禁止直接报裸隐含概率。
- **让球方向**使用数据库中文表述，**禁止自行换算盘口方向**。负样本：亚盘「客让 0.5」与竞彩「让 +1」是不同盘口，**禁止混在一起表述**。
- **数据缺口标注**按 `foretell-status-dictionary` 用户可见标注（「数据不足」「不适用」），单段一次，**不重复**「⚠️ 数据缺口」式吐槽语气。

## 数据诚实规则

| 情况 | 处理 |
|------|------|
| 欧赔 + 亚盘均缺失 | 整段标注「盘口数据不足，建议跳过此场」 |
| 某维度 DATA_MISSING | 写「数据不足」，不编造 |
| 某维度 NOT_APPLICABLE | 写「不适用」并说明原因 |
| 淘汰赛问联赛排名 | 说明不适用，补充替代信息 |

### 情报维度兜底（intel 全缺失）

`intel-analyst` 返回 `status_map` 全 `DATA_MISSING`（阵容、伤停、情报均缺）时：

1. 主智能体触发 `internet_search("{主队} {客队} 首发 伤停")` **1 次**兜底（工具已在主智能体工具列表）。
2. 提取首发 / 伤停线索补入第④段；仍无结果时第④段标注「情报数据不足，建议关注赛前首发公布」。

### 关键维度全缺失降级

当**情报或盘口任一关键维度全缺失**（且兜底仍无结果）时：

- 主推方向**降级**为「关注赛前首发 / 建议观看」，**不强行给让球主推方向**。
- 第①段置信度标注为「低」，并说明降级原因。
- 禁止在关键维度缺失下仍输出高置信度让球主推。

## 与场景 G 的衔接

完成六段分析后，用户可能追问「比分预测」「哪一队率先进球」——见 `foretell-follow-up`，**无需重新定位**。

## 输出纪律

遵守 `foretell-output-discipline`：不暴露工具名、不输出中间态、购彩结尾加风险提示。
