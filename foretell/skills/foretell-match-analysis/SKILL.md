# 单场深度分析（场景 B）

## 何时使用

用户意图为**多维度整合后的专业判断**，而非轻量事实查询：

| 典型问法 | 说明 |
|----------|------|
| 「分析周二004 巴黎 VS 拜仁」 | 竞彩编号 + 深度分析 |
| 「谁会赢」「全面分析一下」 | 预测/模板分析 |
| 「按四步稳球模板分析」 | 结构化深度输出 |

**不要**与场景 A 轻量查询混用：用户问赛程列表、积分榜时走 `foretell-light-query`。

## 处理流程

```
1. 定位比赛（task → entity-resolver，或主智能体直接 resolve）
2. 并行委派三维度子智能体（单次 task 含完整 match_id 与球队 ID）：
   - fundamentals-analyst → 积分、近况、交锋
   - odds-analyst         → 欧赔、亚盘、大小球、走势
   - intel-analyst        → 阵容、伤停、情报
3. 收集各 task 返回的结构化 JSON
4. 按本 Skill 模板归纳最终输出
5. 写入 Checkpointer（供场景 G 追问）
```

## 子智能体委派要点

- 子智能体**无状态**：每次 task 须包含完整 match_id、team_id、指令
- 可并行发起三个 task，不必串行等待
- 子智能体返回 JSON，主智能体在上下文中汇总，**不落虚拟文件**

## 输出结构（足球默认六段）

1. **核心结论与推荐**（4 行以内：胜平负倾向、让球倾向、大小球倾向、比分区间、置信度）
2. **赛事基础信息概览**（时间、联赛、主客、场地）
3. **赛程体能与环境因素**
4. **伤停与战术权重**
5. **机构操盘与资金陷阱**
6. **战术打法与比赛走势推演**

篮球使用五段式（见 Foretell 产品文档 §2.4）。

## 数据诚实规则

| 情况 | 处理 |
|------|------|
| 欧赔 + 亚盘均缺失 | 整段标注「盘口数据不足，建议跳过此场」 |
| 某维度 DATA_MISSING | 写「数据不足」，不编造 |
| 某维度 NOT_APPLICABLE | 写「不适用」并说明原因 |
| 淘汰赛问联赛排名 | 说明不适用，补充替代信息 |

## 与场景 G 的衔接

完成六段分析后，用户可能追问「比分预测」「哪一队率先进球」——见 `foretell-follow-up`，**无需重新定位**。

## 输出纪律

遵守 `foretell-output-discipline`：不暴露工具名、不输出中间态、购彩结尾加风险提示。
