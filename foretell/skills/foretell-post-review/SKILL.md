# 赛后复盘（场景 C）

## 何时使用

用户关心**已结束比赛**的过程解读，而非赛前预测：

| 典型问法 | 说明 |
|----------|------|
| 「昨天皇马那场怎么回事」 | 完场赛果 + 过程解读 |
| 「复盘一下湖人勇士」 | 篮球完场复盘 |
| 「上周日英超焦点战回顾」 | 按日期 + 联赛定位完场比赛 |

**不要**与场景 B 赛前分析混用：用户问「谁会赢」「怎么买」时走深度分析或购彩路径。

## 处理流程

```
1. 定位已完场比赛（task → entity-resolver，注意系列赛场次约束）
2. 确认 status=finished；未结束则说明无法复盘
3. 调用 get_match_result 获取赛果、技术统计、关键事件
4. 按本 Skill 复盘模板归纳输出
```

## 输出结构（足球）

1. **赛果速览**：最终比分、胜负、半场/全场走势一句话
2. **比赛进程**：按时段（上下半场或节次）叙述关键转折
3. **数据亮点**：射门、控球、角球等关键统计对比（有则写，无则标注数据不足）
4. **关键球员与事件**：进球、助攻、红黄牌、换人影响
5. **赛后影响**：积分榜变化、晋级/淘汰形势（联赛/杯赛适用时）

## 输出结构（篮球）

1. **赛果速览**：最终比分、四节得分分布
2. **比赛进程**：领先交替、关键节次节奏
3. **数据亮点**：命中率、篮板、助攻等对比
4. **关键球员**：得分王、正负值、决定性表现
5. **系列赛语境**（季后赛）：系列赛比分、下一场展望

## 购彩与风险提示

- 复盘**默认不涉及**购彩推荐，**无需**附加投注风险提示
- 用户主动追问「那场该怎么买」「复盘后觉得下一场怎么投」时，切换 `foretell-betting-single` 并加风险提示

## 数据诚实

| 情况 | 处理 |
|------|------|
| 比赛未结束 | 说明无法提供完场复盘，可改答赛前分析（若用户愿意） |
| 赛果数据缺失 | 标注「赛果数据不足」，不编造比分或事件 |
| 技术统计缺失 | 跳过该段或写「统计数据不足」 |

## 输出纪律

遵守 `foretell-output-discipline`：不暴露工具名、不输出「正在查询」。
