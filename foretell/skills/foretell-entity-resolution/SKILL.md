# 实体识别与场次定位

## 何时使用

- 用户提到「A VS B」「湖人勇士今晚」「利物浦热刺这场怎么买」
- 用户给出竞彩编号如「周二004」「周六071」
- 用户询问某队、某联赛、某球员，需先标准化再查数据
- 任何需要先「定位对象」再查询的场景

## 四种提问入口（见 Foretell 产品文档 §4.1）

### 入口一：对阵双方（最高频）

1. 从原文识别两支球队，规范为「A VS B」
2. 剥离非球队名部分（「这场怎么买」「赔率对比」等）
3. 若有日期或相对日期（今天/今晚/明天），换算为明确日期
4. 调用比赛定位获取候选列表，结合日期、赛事、主客队和上下文选择正确 `match_id`
5. 候选不足以判断时说明歧义或请求补充日期/赛事；确认后续单场查询透传运动类型（足球/篮球）

**系列赛场次约束（G7 等）：**

- 用户明确 G7、抢七、第七场、系列赛第 N 场时，**必须保留该约束**
- 未找到指定系列赛场次时如实告知，**禁止**降级分析其他场次
- **禁止**用 G6 结果反驳用户指定的 G7 分析请求

### 入口二：竞彩编号

1. 识别玩法类型（竞彩足球 101 / 竞彩篮球 201 / 十四场 401 / 北单 301 等）
2. 解析场次编号或期号
3. 返回彩票 entry 及可能的比赛候选，结合主客队、日期、赛事选择 `match_id`

**竞彩 `issue_num` 编码（数据库）：**

- 竞彩足球 / 竞彩篮球使用 1 位周几 + 3 位场次序号：`6071` = 周六071，`7301` = 周日301
- 周几：周一=1 … 周六=6，周日=7
- 从用户原文匹配 `周[一二三四五六日]\d{3}` 后查询
- 十四场 / 半全场 / 进球彩使用期号 + 「第 N 场」定位；未给期号时返回候选，避免静默拍板

### 入口三：球队 / 联赛 / 球员

1. 识别实体类型
2. 标准化名称后查询
3. 未找到时触发网络搜索兜底（最多 1 次），再重新定位

**搜索触发条件（任一满足即触发，记录 #006）：**
- `ENTITY_NOT_FOUND`：冷门队名 / DB 无候选
- **DB 结果异常**：场次数与赛制不符（如世界杯只查到 2 场小组赛，48 队制应为 3 场）
- **与用户陈述冲突**：用户明确说「明天有世界杯」但 DB 未返回未赛场
- 流程：换 DB 路径（`direction=all` / 竞彩 / `resolve_match`）→ 仍异常 → `internet_search` 1 次 → 提取队名 + 日期 → 回 DB 重查
- **禁止**：DB 未穷尽前断言「没有比赛」

### 入口四：纯日期或规则

无需定位具体比赛，直接查赛程或解释规则。

### 实体链赛程（某队 + 赛事 + 下一场）

1. `resolve_team` 返回候选列表，结合 `name` / `name_en` / `national` / `aliases` 选择正确 `team_id`（如「葡萄牙」可能同时返回国家队与葡萄牙体育）
2. 若用户提及赛事 → `resolve_league` 返回候选列表，结合日期、国家、赛事语境选择正确 `league_id`
3. `get_team_schedule(team_id, league_id=..., direction=upcoming)`

**direction 强制约束（治假阴性）：**
- 「下一场 / 未来 / 明天 / 今晚未赛 / 接下来」类问法**必须**传 `direction="upcoming"`，**禁止**用默认值 `recent`（`recent` 只查已完场，会漏掉所有未赛场）
- 「最近五场 / 近况」→ `recent`；「所有 / 全部」→ `all`
- 负样本：用默认值 `recent` 查「下一场」→ 只回 2 场完场 → 误判「小组赛已结束 / 明天无赛」（记录 #003/#004）

## 禁止由残缺数据推断赛事阶段（强制）

- **禁止**由「查到 N 场完场」推断「小组赛 / 赛事已结束」
- 赛事阶段（是否淘汰、几场小组赛、是否出局）须来自 DB 积分榜或官方赛制，**不得**凭查到场次数臆断
- 负样本：「两场小组赛都踢完了 → 明天没有比赛」（记录 #004/#005，实际 2026 世界杯 48 队制每组 3 场）
- 未查全时答「未查到第 N 场，已查范围…」，**禁止**「没有比赛 / 小组赛已结束 / 可能已出局」

## 核心原则

**先拿候选与证据，再选择 ID 查询**——避免同名球队、跨日比赛、系列赛多场次导致的张冠李戴。

## 常见误区

| 误区 | 纠正 |
|------|------|
| 未定位直接查数据 | 必须先从候选中确认 match_id |
| G7 被当作普通对阵 | 保留系列赛约束，不降级 |
| 用网页信息替代结构化定位 | 冷门赛事先搜索确认队名再定位 |
