# 实体识别与场次定位

## 何时使用

- 用户提到「A VS B」「湖人勇士今晚」「利物浦热刺这场怎么买」
- 用户给出竞彩编号如「周二004」「周一305」
- 用户询问某队、某联赛、某球员，需先标准化再查数据
- 任何需要先「定位对象」再查询的场景

## 四种提问入口（见 Foretell 产品文档 §4.1）

### 入口一：对阵双方（最高频）

1. 从原文识别两支球队，规范为「A VS B」
2. 剥离非球队名部分（「这场怎么买」「赔率对比」等）
3. 若有日期或相对日期（今天/今晚/明天），换算为明确日期
4. 调用比赛定位，获取唯一 `match_id`
5. 后续单场查询透传运动类型（足球/篮球）

**系列赛场次约束（G7 等）：**

- 用户明确 G7、抢七、第七场、系列赛第 N 场时，**必须保留该约束**
- 未找到指定系列赛场次时如实告知，**禁止**降级分析其他场次
- **禁止**用 G6 结果反驳用户指定的 G7 分析请求

### 入口二：竞彩编号

1. 识别玩法类型（竞彩足球 101 / 竞彩篮球 201 / 十四场 401 / 北单 301 等）
2. 解析场次编号或期号
3. 映射到具体比赛及联赛、主客队

### 入口三：球队 / 联赛 / 球员

1. 识别实体类型
2. 标准化名称后查询
3. 未找到时可触发网络搜索兜底（最多 1 次），再重新定位

### 入口四：纯日期或规则

无需定位具体比赛，直接查赛程或解释规则。

## 核心原则

**先定位，再查询**——避免同名球队、跨日比赛、系列赛多场次导致的张冠李戴。

## 常见误区

| 误区 | 纠正 |
|------|------|
| 未定位直接查数据 | 必须先定位 match_id |
| G7 被当作普通对阵 | 保留系列赛约束，不降级 |
| 用网页信息替代结构化定位 | 冷门赛事先搜索确认队名再定位 |
