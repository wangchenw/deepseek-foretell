# 赛事前瞻 / 争冠保级（场景 H）

## 何时使用

联赛**宏观层面**的判断性问题：

| 典型问法 | 说明 |
|----------|------|
| 「英超争冠还有悬念吗」 | 争冠形势推演 |
| 「谁已经降级了」 | 降级区判定 |
| 「哪些球队锁定了季后赛」 | NBA 季后赛席位 |
| 「皇马已经锁定冠军了吗」 | 冠军是否数学锁定 |

**不要**与单场分析混用；不涉及具体对阵深度六段式（除非用户追加）。

## 处理流程

```
1. 识别联赛（resolve_league）
2. get_standings 获取积分榜
3. 对争冠/保级候选球队 get_team_schedule 查看剩余赛程（按需）
4. 结合积分差、剩余场次做数学阈值推理
5. 按本 Skill 叙事模板输出
```

## 语义精度要求（强制）

| 表述 | 含义 | 对外怎么说 |
|------|------|------------|
| **锁定排名** | 精确名次已确定，再无变化可能 | 「已锁定第 X 名」或「已夺冠/已降级」 |
| **锁定不低于某排名** | 名次下限确定，具体名次仍可能变 | 「至少锁定第 X 名」，**不说**「已锁定第 X 名」 |
| **仍有悬念** | 数学上仍存在多种结果 | 说明领先分差、剩余场次、关键对手 |

**禁止**把「至少第 X 名」说成「已锁定第 X 名」。

## 输出结构

1. **形势总览**：联赛阶段、榜首/榜尾格局一句话
2. **争冠组**（适用时）：积分差、剩余场次、关键对决、是否仍有悬念
3. **保级/降级区**（足球适用时）：降级线附近球队、分差、形势
4. **季后赛/席位**（篮球适用时）：分区排名、胜场差、锁定情况
5. **数据依据**：引用积分榜关键行，区分事实与推断

## 数据诚实

| 情况 | 处理 |
|------|------|
| 积分榜缺失 | 说明数据不足，不编造排名 |
| 淘汰赛阶段问联赛排名 | NOT_APPLICABLE，说明不适用并给替代信息 |
| 剩余赛程未知 | 降权推理，标注不确定性 |

## 购彩与风险提示

- 宏观前瞻**默认不涉及**购彩，无需投注风险提示
- 用户追问「基于这个买哪场」→ 切换单场或批量购彩 Skill

## 输出纪律

遵守 `foretell-output-discipline`：不暴露工具名、不输出中间态。
