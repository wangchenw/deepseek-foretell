# 轻量查询（场景 A）

## 何时使用

用户意图为**事实性、列表性**查询，无需深度分析：

| 类型 | 典型问法 |
|------|----------|
| 按日期查赛程 | 「今天有什么足球比赛」「今晚 NBA 赛程」 |
| 按球队查赛程 | 「利物浦接下来打谁」「湖人最近五场」 |
| 按彩票查赛程 | 「今天竞彩足球有哪些」「本期十四场」 |
| 积分榜 / 近况 | 「英超积分榜」「利物浦最近五场」 |
| 单场事实 | 某场比分、某球员数据 |

## 三条赛程路径（不可互换）

见 Foretell 产品文档 §2.3：

1. **按日期** → `get_schedule_by_date`
2. **按球队** → 先 `resolve_team`，再 `get_team_schedule`
3. **按彩票** → `get_lottery_schedule`

三者不可互换：按日期查不能替代「某队接下来打谁」。

## 处理流程

1. 若涉及具体实体，先定位（见 `foretell-entity-resolution`）
2. 单次或少量工具调用获取数据
3. 归纳后直接回答，**不**走六段式深度模板
4. 遵守 `foretell-output-discipline` 与 `foretell-status-dictionary`

## 与深度分析的分轨

| 轻量 | 深度 |
|------|------|
| 赛程列表、积分榜、单场比分 | 预测、复盘、模板分析、盘口解读 |
| 单次查询 → 归纳回答 | 多维度并行 → 六段式输出 |

用户问「分析」「预测」「怎么买」时，**不要**走轻量路径。

## 输出要求

- 条理清晰，适合球迷快速扫读
- 数据缺失时按状态码标注，不编造
- 列表类结果优先用简洁条目呈现
